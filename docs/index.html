EOA Crypt

Proof of Concept for TokeyoWeb3Hackathon

ヘルプページ

  • 現在はPoC段階です。実際の重要なデータに対しては使用しないでください。
  • 重要な資産を管理しているアドレスでの登録は避けてください。
  • publicKeyの登録やメッセージの送信の際にはテストネット(GoerliETH)のガス代がかかります。予めご了承ください。

ウォレットを接続し暗号化メッセージを送信する（共有、保存）

publicKeyを登録する(初回のみ）
  1. [connect wallet]ボタンを押し、ウォレットを接続します。
  2. publicKeyをオンチェーンに登録する為の以下の要求が行われます。（初回のみ）
    1. ウォレットをEOA Cryptに接続する許可要求
    2. EOA Cryptだけで内部利用するprivateKeyおよびpublicKeyを作成する為の署名要求
    3. 作成されたpublicKeyをオンチェーンへ保存する為のトランザクション要求
  3. 画面右上に自分のアドレスが表示されていることを確認します。
  4. 送信先欄に自分のアドレス（省略表示にて）が表示されていることを確認します。
  5. 緑色で表示されているログを確認します。

  1. ウォレットを接続します。

他人と共有したいテキストを暗号化してIPFSに保存する
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. 送信先欄に自分のアドレスの省略表現が表示されていることを確認します。
  4. 送信先欄をクリックし、送信したい相手のアドレス（省略表記）を選択して送信先欄に追加します。
  5. [エディタ]に暗号化して共有したいテキストを入力します。
  6. [送信]ボタンを押し、署名を行って送信します。
  7. 緑色で表示されているログを確認します。

送信（共有）された暗号化テキストを復号する

自分宛の暗号テキストを復号する
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. [受信履歴]欄まで下にスクロールします。
  4. [自分宛のみ]を選択します。
  5. [復号して表示する]をクリックします。
  6. 復号の為の署名を行います。
  7. [エディタ]に復号されたテキストが表示されます。
他人宛の暗号テキストの復元を試してみる
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. [受信履歴]欄まで下にスクロールします。
  4. [全ての宛先]が選択されていることを確認します。
  5. 外枠がついていない（自分宛ではない）受信履歴を探します。
  6. その受信履歴の[復号して表示する]をクリックします。
  7. 復号できない旨のアラートが表示されることを確認します。
  8. 緑色で表示されているログを確認します。

暗号化メッセージをファイルに保存する

受信履歴からメッセージをダウンロードする
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. [受信履歴]欄まで下にスクロールします。
  4. 保存したいメッセージを探します。
  5. その受信履歴の[ダウンロード]をクリックします。
  6. 保存場所を選択し、ファイルとして保存します。
ローカルに保存したファイルをアップロードして復号する
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. [エディタ]のアップロードをクリックして復号したいファイルを選択します
  4. 暗号化された状態のテキストが[エディタ]に表示されます。
  5. [エディタ内容 復号]をクリックします。
  6. 復号の為の署名を行います。
  7. 復号された内容を確認します。

任意のテキストを暗号化してローカルに保存する

自分しか見ることができないテキストを暗号化してローカルに保存する
  1. ウォレットを接続します。
  2. 画面右上に自分のアドレスが表示されていることを確認します。
  3. [エディタ]に暗号化して保存したいテキストを入力します。
  4. [エディタ内容 暗号化]をクリックします。
  5. 暗号化されたことを確認し、[ダウンロード]をクリックします。
  6. 保存場所を選択し、ファイルとして保存します。

その他の利用方法 / Tips

受信履歴からIPFS上の暗号データの内容を確認する
  1. ウォレットを接続します。
  2. 各メッセージの[保存先]に記載されているリンクをクリックします。

補足事項

  • 自分宛以外の暗号データも確認することができます。
  • 暗号データを[エディタ]にコピペした後、[エディタ内容 復号]を使って復号することもできます。
受信履歴からメタデータ（復号プロセス途中のJSON）の内容を確認する
  1. 受信履歴のメッセージの「メタデータ表示」ボタンを押す。
  2. お好みでエディタ上の「入力エリアを折り返す」のオン/オフを切り替えます。

補足事項

  • この機能はデータ構造の理解を助けることを目的として、復号プロセスの途中経過をDemo用に表示しています。
  • 自分宛以外のメッセージのメタデータも確認することができます。
  • メタデータを表示させた状態から、[エディタ内容 復号]を使って復号することもできます。
自分宛以外のメッセージの復号を試してみる
  1. IPFS上のテキストや[メタデータ表示]の機能を利用して得たメタデータをいじり倒して気合と根性で復号します。
  2. バグや脆弱性の検出にご協力ください。
